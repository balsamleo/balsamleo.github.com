精通Javascript笔记

Pro Javascript Techniques

读书笔记，章节名是我自己理解再重命名的，基于我对本书和Javascript的理解，是笔记就以自己的理解为重。

1.第一章：认识Javascript

1.DOM

DOM 是表达XML文档最常见的方式，应用最为广泛，大部分Web语言都提供了相应的实现。DOM给开发者提供了一种定位XML层级结构的直观方法。 HTML只是XML的一个子集。Javascript中绝大部分的操作都是Javascript和网页不同HTML元素之间的交互，DOM简化了这种交互。

2.事件

现代Javascript程序既要展示数据，也要有良好的数据展示效果，可以通过给事件注册函数达到这种要求。

###　3.Ajax

所有的现在浏览器都支持XMLHTTPRequest，Ajax的时代就到了。

2.第二章：基于对象的Javascript

引用Reference，作用域Scope，闭包Closure，上下文Context是本章重点。

1.作用域

Javascript的作用域由函数划分，而不是块作用域(比如while,if和for语句)。

2.闭包

内层函数可以引用上层函数内的变量，即便上层函数已经终止执行。

编写自执行的匿名函数可以把原本属于全局的变量都隐藏起来，避免多余的全局变量影响其他库。

更详细了解闭包的概念，参考文章

3.上下文（执行环境）

Javascript中，代码总有一个上下文对象（代码处于该对象内）。上下文对象通过this变量体现，这个变量永远指向当前代码所处对象内。

4.对象

对象是Javascript的基础，这门语言所有东西都是对象。对象是一系列属性的集合，和其他语言中的散列表结构类似。
1.公有方法，原型链上
2.私有方法，暂不理解
3.特权方法，私有方法和公有方法的混合体
4.静态方法

6.第六章：事件

Javascript事件模型是相当独特的。不适用任何线程，完全是异步的。程序中的代码由于其他动作，比如用户点击或页面加载才会触发。

线程程序设计和异步程序设计的根本区别在于如何等待事件的发生。线程程序中要不断地检查条件是否满足。异步程序中则使用事件处理函数注册一个回调函数，一旦事件触发，该事件处理函数就会运行回调函数

因为while循环的持续运行并阻塞应用程序的其他普通流程，它永远无法达到true值。结果只能是用户浏览器暂停，延迟甚至崩溃。
一个DOM元素触发某个特定事件的时候，指定一个回调函数来处理它。也就是说为需要执行的代码提供一个引用，让浏览器去处理剩下的细节。

下面的例子是将封装函数绑定到事件处理函数上(window.oonload),window.onload()在页面加载完毕后被调用。点击事件/鼠标悬停事件也是如此。

取决于事件类型和元素在DOM中的位置，事件执行顺序和处理会不同。Javascript事件在两个阶段中执行：捕获和冒泡。
这一节比较复杂，需要再专门了解。理解了Javascript中的事件，就对Javascript的认识更深刻了。
