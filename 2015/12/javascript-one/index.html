Javascript学习1219

  • 30天js

1.键值对/对象

键值对：键可能多次定义，但在特定context中只有唯一的一个值。值本身可能也是键值对。
对象：键值对的集合

Address是一个对象

2.全局作用域，全局对象

js代码一旦执行，就产生了‘全局对象’和‘this’

在浏览器中 this===window===frames

3.执行上下文，变量提升

变量提升的原因:

Creation Phase(创建阶段)：
‘this’,外部环境，为变量和函数申请内存

在代码执行前，变量和函数在内存中已经存在了，代码的执行是一行一行的，a已经在内存中存在了，但是解释器不知道变量a的具体内容，所以显示’undefined’，如果是a不定义，则显示’Uncaught ReferenceError’(因为a在内存中不存在)

Javascript中，初始化值都是undefined,undefined是一种特殊值

Execution Phase(执行阶段):
链接-编译-解释-执行

4.函数调用和Execution Stack(执行栈)

使用()来调用函数，将调用的函数放在执行栈的最上方，执行完毕弹出栈。

5.函数，上下文，变量环境,作用域(scope)

变量环境：变量存在的内存空间，在内存中的相互关系

因为先输出全局的myVar=1,然后是a()中的myVar=2,然后是b()中的myvar(undefined).

6.作用域链(The Scope Chain)

从词法上来说，b()是在全局环境之上的，与a()同级。b()的外部环境是全局环境，所以它会去获取全局变量myVar的值，也就是1.

7.作用域,ES6和let

代码块

8.回调

链接器/网络请求/引擎，之间的交互是异步的，只是引擎中的js是同步执行的。

所有js代码执行之后，引擎会去监听事件队列，按顺序执行队列中的事件。（3秒内点击产生clickHandler，也是同样的执行顺序，3秒之后点击产生clickHandler则是马上执行，因为堆栈以及为空，可以处理事件队列）

9.Javascript数据类型

动态类型，引擎会在执行代码时自己判断

6种基本数据类型（不是对象）

1.undefined,变量未赋值
2.null, 变量值未null
3.boolean, true和false
4.Number，js只有浮点数数字类型
5.string，字符串 ‘’,””都可以用
6.symbol，ES6中可用
